Projet Informatique Théorique 2015 Documentation

Modalités

Le projet est à réaliser en C par groupes de 2 personnes maximum. L'utilisation de la bibliothèque fournie est obligatoire. Il est à rendre en deux parties. Contrairement au projet de l'UE AS&PP3, il n'y aura pas de soutenance et on ne demande pas de rapport, mais le code doit être commenté. La notation tiendra compte du degré de réalisation du sujet, de la qualité du code (portabilité, lisibilité, documentation), et du jeu de tests.

La programme est à rendre au format archive tar compressée par gzip, sous le nom nom1-nom2.tgz par mail à l'adresse it2@l.nosp@m.abri.nosp@m..fr. La décompression de l'archive doit fournir un répertoire nom1-nom2 dans lequel se trouveront seulement 2 fichiers: automate.c et rationnel.c.

Attention
Vous devez utiliser un système de gestion de projets, comme subversion (savanne au cremi) ou git. Les dates limites de remise du projet par mail sont strictes :
  • 3 avril 2015, 12h00 pour la première partie.
  • 7 mai 2015, 12h00 pour la deuxième partie.

Objectif

La première partie concerne la minimisation d'automates et le calcul de l'automate de Glushkov à partir d'une expression rationnelle. La seconde partie est la programmation de l'algorithme de calcul d'une expression rationnelle en utilisant le lemme d'Arden. Le projet consiste à écrire des fonctions implémentant ces algorithmes en complétant deux fichiers tous deux fournis dans cette archive:

  • automate.c, pour la fonction miroir() et la fonction creer_automate_minimal() de minimisation.
  • rationnel.c, pour toutes les autres fonctions.

Aucun autre fichier ne doit être modifié (les autres seront remplacés par les fichiers originaux lors de l'évaluation). Vous devez aussi utiliser le fichier makefile fourni. Les fichiers d'interface automate.h et rationnel.h contiennent les prototypes des fonctions, y compris celles à implémenter. Dans les fichiers automate.c et rationnel.c, le code des fonctions à écrire est remplacé par la macro A_FAIRE ou A_FAIRE_RETURN(). Le code nécessaire est commenté, et la documentation est disponible via ces pages web. La liste des fonctions à implémenter est aussi disponible ici, ou par l'onglet Pages Associées ci-dessus.

Tests

Des tests sont fournis dans le répertoire ./tests pour certaines fonctions, comme test_premier.c qui teste la fonction premier(). Pour les lancer, il suffit d'utiliser make check, qui compile les tests, les lance et affiche le nombre de tests réussis et le nombre de tests échoués. Il est fortement conseillé d'ajouter vos propres tests car nous utiliserons un fichier plus complet lors de l'évaluation (un pour chaque fonction). Il suffit pour cela de créer un nouveau fichier par test dans le répertoire ./tests, en suivant le motif des fichiers de test déjà écrits.

Note
Comme vous devez écrire la plupart des fonctions testées, il est normal que les tests échouent pour l'instant, sauf le test de déterminisation d'un automate (la fonction de déterminisation vous est fournie).
