Description détaillée

Le type décrivant une expression rationnelle.

Une expression contient:

  • un champ étiquette de type Noeud, donnant le type de l'expression. (l'ensemble vide peut être représenté par NULL, si besoin).
  • un caractère, dans le cas où l'expression est un noeud LETTRE.
  • une position_min, et une position_max.

Documentation des champs

struct Rationnel* Rationnel::gauche

Fils gauche, utilisé pour les noeuds binaires et l'étoile:

  • Si le noeud est de type UNION ou CONCAT, représente la sous-expression gauche.
  • Si le noeud est de type STAR, représente la sous-expression, "sous" l'étoile.
  • NULL si le noeud est de type EPSILON ou LETTRE.

Définition à la ligne 63 du fichier rationnel.h.

struct Rationnel* Rationnel::droit

Fils droit, utilisé pour les noeuds binaires seulement:

  • Si le noeud est de type UNION ou CONCAT, représente la sous-expression droite.
  • NULL si le noeud est de type EPSILON, LETTRE ou STAR.

Définition à la ligne 69 du fichier rationnel.h.

int Rationnel::position_min

Position utilisée pour l'algorithme de Glushkov:

  • Pour une lettre, désigne la position de la lettre, en commençant la numéroration par 1. Pour une lettre, position_min a donc la même valeur que position_max.
  • Pour un noeud interne, désigne la position la plus petite d'une lettre de la sous-expression correspondant à ce noeud.
  • Non utilisée pour EPSILON.

Définition à la ligne 74 du fichier rationnel.h.

int Rationnel::position_max

Position utilisée pour l'algorithme de Glushkov:

  • Pour une lettre, désigne la position de la lettre, en commençant la numéroration par 1. Pour une lettre, position_max a donc la même valeur que position_min.
  • Pour un noeud interne, désigne la position la plus grande d'une lettre de la sous-expression correspondant à ce noeud. Par exemple, pour l'expression \((a+b)^*\) les champs position_min et position_max valent tous les deux 1 pour la sous-expression \(a\), tous les deux 2 pour la sous-expression \(b\), mais position_min vaut 1 et position_max vaut 2 pour la sous-expression \(a+b\), ainsi que pour l'expression complète \((a+b)^*\).
  • Non utilisée pour EPSILON.
