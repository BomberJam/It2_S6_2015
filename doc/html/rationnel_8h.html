Documentation des définitions de type

typedef enum Noeud Noeud

Type d'expression.

Les différents types pour une expression sont:

  • EPSILON (mot vide),
  • LETTRE (une lettre),
  • STAR (l'étoile d'une sous-expression)
  • UNION (l'union de 2 sous-expressions)
  • CONCAT (la concaténation de 2 sous-expressions)

S'il est nécessaire de représenter l'expression vide, on utilise NULL comme expression rationnelle (voir Rationnel).

typedef struct Rationnel Rationnel

Le type décrivant une expression rationnelle.

Une expression contient:

  • un champ étiquette de type Noeud, donnant le type de l'expression. (l'ensemble vide peut être représenté par NULL, si besoin).
  • un caractère, dans le cas où l'expression est un noeud LETTRE.
  • une position_min, et une position_max.
typedef Rationnel*** Systeme

Un système d'équations est représenté par un tableau à 2 dimensions. Chaque case du tableau contient un pointeur sur un Rationnel.

Par exemple, le système

\begin{alignat*}{2} X_0 &=\ & a\cdot X_0 +\ &b\cdot X_1 + \varepsilon\\ X_1 &=&& c\cdot X_1 + d^* \end{alignat*}

sera représenté par le tableau systeme à 2 lignes et 3 colonnes contenant les pointeurs sur les Rationnel suivants:

\begin{array}{|c|c|c|} \hline a & b & \varepsilon\\ \hline \emptyset & c & d^*\\ \hline \end{array}

Ainsi, systeme[0][2] contient un pointeur sur l'expression rationnelle \(\varepsilon\), et systeme[1][0] contient le pointeur NULL représentant l'expression \(\emptyset\).

Rationnel* expression_to_rationnel ( const char *  expr)

Construit le rationnel correspondant à une expression expr, donnée sous forme d'une chaîne de caractères.

La syntaxe pour les expressions est la suivante:

  • les lettres sont données en minuscule.
  • la concaténation se note par un point '.'
  • l'union se note par '+'.
  • l'étoile se note par '*'.
  • on peut parenthéser une sous-expression avec les parenthèses '('...)'. Le parseur ne prend pas en compte le mot vide ni le langage vide.
    Paramètres
    exprexpression rationnelle donnée avec la syntaxe ci-dessus.

Rationnel** resoudre_variable_arden ( Rationnel **  ligne,
int  numero_variable,
int  nb_vars 
)
A faire:
Résout une équation de langages en utilisant le lemme d'Arden: si \(U\) et \(V\) sont deux langages et \(U\) ne contient pas le mot vide, et si \(X\) est un langage tel que \(X=UX+V\), alors \(X=U^*V\).
Paramètres
ligneLa ligne codant l'équation de langages.
numero_variableLe numéro de la variable jouant le rôle de X dans le lemme d'Arden, à exprimer en fonction des autres variables. Si ce numéro est \(i\), le membre gauche de l'équation est donc \(X_i\) (ce membre gauche n'est pas codé dans le paramètre 'ligne').
nb_varsLe nombre de variables.
Renvoie
Une ligne codant l'équation résultat, dans laquelle la variable de numéro 'numero_variable' a été exprimée en fonction des autres.

Par exemple, si 'nb_vars' vaut 3, il y a 3 variables \(X_0\), \(X_1\) et \(X_2\). Si en plus l'argument 'numero_variable' vaut 1 et l'argument 'ligne' vaut \(\{a, b, \sf{NULL},\varepsilon \}\), alors l'équation représentée par ces arguments est

\[X_1 = a\cdot X_0 + b\cdot X_1 + \emptyset\cdot X_2 + \varepsilon\]

ou encore

\[X_1 = a\cdot X_0 + b\cdot X_1 + \varepsilon.\]

Le lemme d'Arden donne

\[X_1 = b^*(a\cdot X_0+\varepsilon)=b^*a\cdot X_0+b^*.\]

La fonction doit donc retourner la ligne

\[\{b^*a, {\sf NULL}, {\sf NULL}, b^* \}.\]
